# SDD Guide

This guide teaches the concepts and operating model behind spec-driven development.

## Learning Path

1. [Foundations](01-foundations.md) - definitions, maturity levels, and why SDD matters now.
2. [Workflow](02-workflow.md) - the lifecycle from problem framing to spec maintenance.
3. [Writing Great Specs](03-writing-great-specs.md) - requirement IDs, acceptance criteria, EARS patterns, and spec quality.
4. [AI Agent Workflow](04-ai-agent-workflow.md) - how to use specs to constrain agent work.
5. [Adoption Playbook](05-adoption-playbook.md) - a 30/60/90-day rollout path.
6. [Glossary](06-glossary.md) - core terms and definitions.
7. [When Not to Use SDD](07-when-not-to-use-sdd.md) - how to avoid over-specifying simple work.
8. [Agent Instructions and SDD Skills](08-agent-instructions-and-skills.md) - how to use `AGENTS.md`, task briefs, and reusable skills together.
9. [Multi-Agent SDD](09-multi-agent-sdd.md) - how to coordinate clarifier, planner, test, implementer, and reviewer agents.
10. [Human-in-the-Loop SDD](10-human-in-the-loop.md) - where human judgment belongs in SDD and orchestration.

Use the [decision ladder](../decision-ladder.md) alongside this guide to decide how much rigor a project needs.
