# Technical Plan Readiness Checklist

A technical plan is ready for tasking when:

- [ ] Architecture is described.
- [ ] System boundaries are clear.
- [ ] Data model changes are listed.
- [ ] API or interface changes are listed.
- [ ] Integration behavior is defined.
- [ ] Failure handling is defined.
- [ ] Security controls are mapped to implementation.
- [ ] Observability plan exists.
- [ ] Migration plan exists if needed.
- [ ] Rollback plan exists if needed.
- [ ] Tradeoffs are documented.
- [ ] Risks have mitigations.
- [ ] No major requirement is unplanned.
