# Accessibility Checklist

For user-facing UI:

- [ ] Keyboard interaction is defined.
- [ ] Focus behavior is defined.
- [ ] Validation errors are associated with fields.
- [ ] Status messages are announced to assistive technologies.
- [ ] Color is not the only means of conveying information.
- [ ] Required fields are programmatically identified.
- [ ] Touch and click targets are usable.
- [ ] Motion and animation are avoidable or limited.
- [ ] Content meets the target WCAG level.
