# SDD Code Review Checklist

During review, ask:

- [ ] Does the PR satisfy the linked requirement IDs?
- [ ] Are all relevant acceptance criteria covered?
- [ ] Are tests tied to requirements?
- [ ] Did the PR change anything outside task scope?
- [ ] Did it introduce a new dependency?
- [ ] Did it change architecture without an ADR or spec update?
- [ ] Did it change data behavior or retention?
- [ ] Did it log sensitive data?
- [ ] Did it preserve accessibility behavior?
- [ ] Did it update docs or specs where behavior changed?
- [ ] Did it include operational visibility where needed?
