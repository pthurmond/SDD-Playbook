# Human-in-the-Loop Checklist

Use this before an agent or team moves to the next SDD stage.

## Decision Ownership

- [ ] The human decision owner is named.
- [ ] The decision needed is explicit.
- [ ] The agent recommendation is labeled as a recommendation, not a decision.
- [ ] Options and tradeoffs are visible when there is more than one valid path.

## Evidence

- [ ] Requirement IDs affected by the decision are listed.
- [ ] Relevant spec, plan, task, or test files are linked.
- [ ] Open questions are classified as blocking, deferred, host choice, or resolved.
- [ ] Risks are listed with mitigations or accepted owners.
- [ ] Tests, checks, or review evidence are listed where applicable.

## Stop Conditions

- [ ] Data, security, privacy, money, compliance, or user-trust changes have human approval.
- [ ] Architecture changes have human approval.
- [ ] Dependency additions have human approval.
- [ ] External contract changes have human approval.
- [ ] Release or rollback risk has human approval.

## Handoff

- [ ] The next agent or person has a bounded task brief.
- [ ] Scope and non-goals are explicit.
- [ ] Validation commands are explicit.
- [ ] Expected output format is explicit.
- [ ] Spec drift is documented or ruled out.
