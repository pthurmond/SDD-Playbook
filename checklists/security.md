# Security Checklist

For security-sensitive features:

- [ ] Authentication requirements are defined.
- [ ] Authorization rules are defined.
- [ ] Input validation is defined.
- [ ] Output encoding is defined where relevant.
- [ ] Sensitive data is classified.
- [ ] Logging excludes secrets and sensitive PII.
- [ ] Secrets are not stored in code or specs.
- [ ] Dependency additions are reviewed.
- [ ] Rate limiting or abuse controls are defined.
- [ ] Audit logging is defined.
- [ ] Error messages do not leak sensitive details.
- [ ] Data retention rules are defined.
- [ ] Encryption requirements are defined.
