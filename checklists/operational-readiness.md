# Operational Readiness Checklist

Before release:

- [ ] Logs identify important events.
- [ ] Metrics exist for success, failure, and latency.
- [ ] Alerts exist for critical failures.
- [ ] Runbook exists for support.
- [ ] Rollback path is documented.
- [ ] Feature flag exists if needed.
- [ ] Data migration is tested.
- [ ] Third-party failure behavior is tested.
- [ ] Dashboards are updated.
- [ ] Support stakeholders know what changed.
