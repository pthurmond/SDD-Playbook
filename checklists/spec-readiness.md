# Spec Readiness Checklist

A spec is ready for technical planning when:

- [ ] The problem statement is clear.
- [ ] The affected users or stakeholders are named.
- [ ] Goals are measurable.
- [ ] Non-goals are explicit.
- [ ] Functional requirements have IDs.
- [ ] Business rules have IDs.
- [ ] Non-functional requirements are measurable.
- [ ] Security and privacy requirements are documented.
- [ ] Accessibility requirements are documented where user-facing UI exists.
- [ ] Operational requirements are documented.
- [ ] Edge cases are listed.
- [ ] Acceptance criteria exist for major requirements.
- [ ] Open questions are documented.
- [ ] Dependencies are named.
- [ ] Assumptions are visible.
- [ ] Stakeholders have reviewed it.
